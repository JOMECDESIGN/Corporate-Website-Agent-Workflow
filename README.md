# Corporate-Website-Agent-Workflow

A lightweight AI-agent workflow for planning, writing, designing, building, and launching a corporate technology website.

This workflow is inspired by the open-source agency-agents repository and adapts selected agents for corporate website development, including brand strategy, content planning, SEO, UX architecture, UI design, frontend implementation, technical writing, project management, and launch validation.

Minimal Description

An AI-agent workflow for building a high-quality corporate technology website.

Purpose

This repository organizes a practical agent-based workflow for creating a company website that is clear, credible, visually consistent, technically feasible, SEO-ready, and easy to maintain.

It is especially suitable for technology companies, AI companies, industrial intelligence companies, smart cabin solution providers, AI-IoT platform companies, intelligent vessel solution providers, and system integration businesses.

Suggested Website Structure

A corporate technology website should not only introduce the company. It should also explain what the company does, why it matters, what capabilities it has, what projects prove those capabilities, and how customers or partners can engage.

Recommended website sections:

Home
About
Core Capabilities
Solutions
Products / Platforms
Case Studies
Technology Insights
Partners / Ecosystem
News
Contact
Agent Workflow
1. Brand Strategy

Responsible agent: Brand Guardian

Outputs:

Brand positioning
Mission and vision statement
Brand tone of voice
Homepage core message
Value proposition
Messaging hierarchy
Visual direction keywords

Key question:

What should the company be remembered for after a visitor spends 30 seconds on the website?

2. Website Content Strategy

Responsible agent: Content Creator

Outputs:

Website sitemap
Homepage copy
Company introduction
Solution page copy
Product platform descriptions
Case study structure
CTA copy
News and insight content framework

Key question:

What should each visitor understand, trust, and do after reading the website?

3. SEO Strategy

Responsible agent: SEO Specialist

Outputs:

Keyword list
Page-level SEO titles
Meta descriptions
Search intent mapping
Internal linking plan
Technical SEO checklist
Structured content suggestions

Key question:

What keywords should this website be found for, and which pages should rank for them?

4. UX Architecture

Responsible agent: UX Architect

Outputs:

Information architecture
Navigation structure
User journey map
Page hierarchy
Conversion path
Mobile-first content flow

Key question:

Can different visitors quickly find what matters to them?

Main visitor groups:

Potential customers
Strategic partners
Government or industry organizations
Investors
Technical evaluators
Talent candidates
5. UI Design

Responsible agent: UI Designer

Outputs:

Visual direction
Design system
Color palette
Typography rules
Component library
Homepage wireframe
Responsive layout guidance
Interaction and animation direction

Key question:

Does the website visually express a serious, innovative, high-technology company?

6. Frontend Development

Responsible agent: Frontend Developer

Outputs:

Frontend architecture
Page templates
Responsive implementation
Component implementation
Performance optimization
Basic accessibility support
Deployment-ready website

Recommended stack:

Next.js, Astro, or Vite + React
TypeScript
Tailwind CSS
MDX or headless CMS for content
Static-first deployment
SEO-friendly routing

Key question:

Is the website fast, stable, responsive, SEO-friendly, and easy to update?

7. Technical Writing

Responsible agent: Technical Writer

Outputs:

Technology capability descriptions
Product platform explanations
System architecture copy
Technical case study copy
Clear terminology
Customer-facing technical language

Key question:

Can non-technical visitors understand the company’s technical value without losing credibility with technical experts?

8. Project Management

Responsible agent: Senior Project Manager

Outputs:

Task breakdown
Milestone plan
Page delivery checklist
Acceptance criteria
Responsibility matrix
Risk list
Launch plan

Key question:

Can the team execute the website without scope creep, unclear ownership, or endless revisions?

9. Launch Validation

Responsible agent: Reality Checker

Outputs:

Pre-launch QA checklist
Content completeness review
Mobile experience review
Broken link check
SEO basics check
Performance check
Contact form validation
Analytics readiness check

Key question:

Is the website truly ready to go live, or does it only look ready?

Quality Checklist

Before launch:

Homepage clearly explains who the company is and what it does

Core capabilities are easy to understand

Solutions are organized by customer scenario

Product or platform pages have clear value and technical credibility

Case studies prove real delivery capability

Navigation works on desktop and mobile

Contact path is obvious

SEO title and meta description exist for each page

Images are optimized

Website loads quickly

No broken links

Contact form works

Analytics is installed

Website can be updated without redesigning everything
